## Architecture Summary

- **Pattern:** Feature-Sliced Design (Next.js app) + layered DAL (handlers/services/repository/domain) + handlers/services/clients/models for Lambdas + envs+modules for Terraform.
- **Enforcement:** ESLint `boundaries` plugin with explicit dependency matrix; `no-restricted-imports` blocks Prisma client at HTTP boundary.
- **Slices:** 14 features under `features/`, 6 widgets, 9 shared/lib subdomains, 8 DAL repository domains, 13 DAL service domains.
- **Largest source file:** 857 LOC (`library-repository.ts`); none exceed 1000 LOC.
- **Notable hotspots (consider future decomposition, not current findings):** `lambdas-py/src/lambdas/services/database.py` (769), `handlers/database_import.py` (696), `features/steam-import/ui/imported-games-list.tsx` (683).
- **Minor boundary leaks:** 2 type-only imports of repository types from feature UI (`features/social/ui/followers-list.tsx`, `following-list.tsx`) ‚Äî type-only, low risk; could be relocated to `features/social/types.ts` or DAL public type re-exports.

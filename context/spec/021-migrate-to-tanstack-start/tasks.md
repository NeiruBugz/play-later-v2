# Tasks: SavePoint Foundation Replacement (Migrate to TanStack Start)

- **Functional Spec:** [functional-spec.md](./functional-spec.md)
- **Technical Spec:** [technical-considerations.md](./technical-considerations.md)
- **Status:** Draft

> **Methodology**:
> - **TDD (red ‚Üí green ‚Üí refactor)**: every slice lists test sub-tasks **before** implementation sub-tasks. A slice is not green until its tests were authored failing first, then made to pass. PR descriptions must reference the failing-test commit.
> - **FSD (Feature-Sliced Design)**: mirror `savepoint-app/`'s layered architecture. Layers (top ‚Üí bottom, lower may not import upper):
>   - `src/app/` ‚Äî providers, root router wiring, global styles, error boundary
>   - `src/routes/` ‚Äî TanStack file-based routes (thin loaders/components ‚Äî pages layer equivalent)
>   - `src/widgets/` ‚Äî composite UI blocks (header, profile composition, library grid)
>   - `src/features/` ‚Äî user-intent slices (`auth-cognito-sign-in`, `edit-profile`, `add-game`, `manage-library-entry`, `journal-compose`, `command-palette`, `follow-user`, ‚Ä¶). Each feature has `model/`, `api/` (server fns), `ui/`
>   - `src/entities/` ‚Äî domain nouns (`session`, `user`, `profile`, `game`, `library-item`, `journal-entry`). Each entity has `model/` (types/zod), `api/` (plain async query fns that throw `AppError`), `ui/` (display-only)
>   - `src/shared/` ‚Äî `lib/` (db, logger, errors, auth-client), `ui/` (shadcn primitives), `config/` (env), `api/` (S3, IGDB low-level clients)
> - **Layer discipline**: server fns live in `features/*/api/`, queries in `entities/*/api/`. A feature server fn composes entity queries; entity queries never import features. Routes import widgets/features; widgets import features/entities; features import entities; entities import shared. No upward imports.
> - **File naming**: **kebab-case** for every file and directory (`cognito-sign-in-button.tsx`, `profile-header.tsx`, `update-profile.server.ts`, `get-game-details.server.ts`, `add-game-modal.tsx`). Exported React component identifiers stay PascalCase; hooks stay `useCamelCase`. Server-only modules use the `.server.ts` suffix. Tests sit next to the file as `<name>.test.ts(x)` (unit) or `<name>.integration.test.ts` (integration). Mirror `savepoint-app/`'s kebab-case convention; CI lint should reject PascalCase filenames.
> - **Discoverability**: every feature ships with the affordance(s) that let a user reach or trigger it from the surfaces it's relevant to. A feature is not GREEN until its trigger (button, link, modal launcher, keyboard shortcut, ‚Ä¶) is rendered and wired on the appropriate surface, and a test asserts the trigger renders and routes/calls the feature. The ‚åòK command palette is a parallel route into features ‚Äî not a substitute for the per-surface CTA. Each slice that introduces a new feature must include explicit `**GREEN (CTA wiring)**` sub-task(s) before the verification gate.
> - **Feedback**: every mutation feature ships with explicit success and error feedback the user can perceive. Default channel is a toast (sonner) for transient confirmation; persistent / form-level errors may additionally render inline via `role="alert"` for screen-reader accessibility. Destructive actions (delete) require a confirmation prompt before firing. A mutation is not GREEN until both success and error paths are wired and a test asserts the toast / alert is fired on the right path. Each slice that introduces a mutation must include explicit `**GREEN (feedback wiring)**` sub-task(s) before the verification gate.
> - Each slice leaves both `savepoint-app/` (untouched) and `savepoint-tanstack/` (under construction) runnable. Verification is local-only until cutover (S20).
> - **Visual parity is split across Slices 14A (done) and 22 (formerly 18A).** Through pre-parity slices the rule is **port-on-demand for shadcn primitives** ‚Äî prefer hand-rolled Tailwind on native elements over speculatively porting a primitive. The initial seven primitives in `src/shared/ui/` (`Button`, `Card`, `Form`, `Input`, `Label`, `Switch`, `Sonner`, plus `Avatar`) cover the work-in-progress surfaces; new primitives are ported only when a slice's *functional* requirement forces it (e.g., `Form` for the settings RHF). Visual / typographic parity with `savepoint-app/` is consolidated late ‚Äî **Slice 14A** discharged the Slice 0‚Äì14 surfaces; **Slice 22** discharges the Slice 15‚Äì21 surfaces ‚Äî so per-slice review focuses on functional + architectural correctness, not pixel matching. Earlier slices may flag missing primitives in `savepoint-tanstack/CLAUDE.md` "Known gaps" without blocking. Rationale: hand-rolled HTML is faster to ship, cheaper to test, and trivially replaceable when a shadcn primitive lands later.
> - **"Port" ‚âÝ "transcribe verbatim."** Canonical (`savepoint-app/`) is the reference for *what features exist*, *what UX patterns the user sees*, and *what file/component vocabulary already names them*. It is **not** a binding implementation contract. Where canonical contains code smells, dead branches, Next-action-shaped wrappers, leaky abstractions, premature splits, redundant Result/Service layers, or patterns that conflict with the tanstack conventions in `savepoint-tanstack/CLAUDE.md` (no Result wrappers, throw-on-error AppError, no `useServerFn`, `.server.ts` is a bundler boundary, etc.) ‚Äî **prefer the better implementation**. Slices may collapse multiple canonical files into one, drop unused props, replace canonical hooks with simpler logic, or skip components entirely when their job is already covered. Each such divergence is logged in `savepoint-tanstack/CLAUDE.md` "Intentional divergences" with a one-line rationale. Verified file lists in slice tasks bound the *scope of UX behavior to cover*, not the literal file count to produce. The slice-6 retro (`.server.ts` rename) and slice-9 deferrals (`distinctByGame`, dynamic platform options) are the established precedent.

Overview

Key improvements to library card design based on UX audit and data model analysis

Current Issues
  • ❌ Fixed column counts create inconsistent card sizes (100-150px)
  • ❌ Actions hidden behind hover (poor discoverability)
  • ❌ Minimal metadata display (only status dot + rating)
  • ❌ No quick access to common actions (log session)
Proposed Solution
  • 180px minimum card width for readable metadata
  • Always-visible CTA for primary actions
  • Rich metadata display using existing database fields
  • Context menu for secondary actions
📊 Data Model Context

All recommendations use existing LibraryItem fields: status, rating, hasBeenPlayed, createdAt, updatedAt, startedAt, actualPlaytime, acquisitionType, platformId. No schema changes required.

✅ Why 180px minimum?
  • Cover legibility: 240px height (3:4 ratio) is comfortable for viewing game art
  • Metadata readable: Platform badges, playtime, rating display without truncation
  • CTA button: "Log Session" fits at 13-14px font size
  • Industry standard: Steam (~190px), Epic Games (~185px), GOG (~180px)
  • Touch targets: 44px minimum (mobile) easily achievable

3. Interactions & Actions

Improved action hierarchy with always-visible primary CTA

Current (Hover-Only)
  • ❌ Actions hidden until hover
  • ❌ Poor discoverability
  • ❌ No mobile equivalent
  • ❌ Requires precise cursor placement
Proposed (Always Visible)
  • Primary CTA always visible (no hover required)
  • Context menu (⋮) for secondary actions
  • Quick rating (click stars directly)
  • Consistent mobile/desktop experience
Action Hierarchy

Primary Action (Always Visible Button)

Status Primary CTA Action
PLAYING "Log Session" Open modal to record playtime
UP_NEXT "Start Playing" Change status to PLAYING
SHELF "Queue It" Change status to UP_NEXT
PLAYED "Replay" Change status to UP_NEXT + set hasBeenPlayed
WISHLIST "Add to Shelf" Change status to SHELF

Secondary Actions (Context Menu ⋮)

  • View Journal Entries
  • Edit Library Details (platform, acquisition type, notes)
  • Change Status (all status options)
  • Remove from Library

Quick Actions (No Modal)

  • Click stars → Update rating instantly (optimistic UI)
  • Click cover → Navigate to game detail page
💡 Implementation Detail: The "Log Session" modal should pre-fill the game and open directly to time input. Use the existing actualPlaytime field (store in minutes).

New Utility Functions
// lib/library-card-helpers.ts export function getContextualDate(item: LibraryItemWithGameDomain): string { const { status, startedAt, createdAt, updatedAt } = item; switch (status) { case LibraryItemStatus.PLAYING: case LibraryItemStatus.UP_NEXT: return startedAt ? `Started ${formatRelative(startedAt)}` : `Added ${formatRelative(createdAt)}`; case LibraryItemStatus.PLAYED: return `Finished ${formatRelative(updatedAt)}`; case LibraryItemStatus.SHELF: case LibraryItemStatus.WISHLIST: default: return `Added ${formatRelative(createdAt)}`; } } export function getPrimaryCTALabel( status: LibraryItemStatus, hasBeenPlayed: boolean ): string { switch (status) { case LibraryItemStatus.PLAYING: return "Log Session"; case LibraryItemStatus.UP_NEXT: return "Start Playing"; case LibraryItemStatus.SHELF: return "Queue It"; case LibraryItemStatus.PLAYED: return hasBeenPlayed ? "Replay" : "Play Again"; case LibraryItemStatus.WISHLIST: return "Add to Shelf"; } }

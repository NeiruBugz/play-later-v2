🎮 Quick Add Flow Redesign

1-click game addition with smart defaults — eliminating the 4-step modal flow

The Problem

Current Flow: Too Many Steps
  • Can't add from search results — user must click through to game detail page
  • 4-step flow: Search → Click result → Detail page → "Add to Library" button → Modal form → Submit
  • No quick-add anywhere — not in navbar, command palette, or search results
  • Friction kills momentum — users browsing/exploring abandon the add flow
Current Flow (4 Steps)
1
Search for game
Type in search, wait for results
↓
2
Click search result
Navigate to game detail page
↓
3
Open "Add to Library" modal
Fill out status, platform, notes
↓
4
Submit form
Game finally added to library

The Solution: 1-Click Quick Add

🎯 Design Principle

Add fast, edit later. Default to smart assumptions (status: UP_NEXT, platform: detect from game metadata), let users refine details after adding. Reduce friction from 4 steps to 1 click.

Smart Defaults
  • Status: UP_NEXT (safest assumption)
  • Platform: Auto-detect primary platform from IGDB metadata
  • Rating: null (user sets after playing)
  • Acquisition: DIGITAL (most common)
  • Notes: Empty (add later if needed)
Where to Add Quick-Add Buttons
  • Search results: Inline "+" button on each card
  • Command palette (⌘K): "Add game to library" action
  • Navbar: Global "Add Game" button
  • Game detail page: Keep existing full modal as fallback
New Flow (1 Step)
1
Click "+" on search result card
Game added with smart defaults → Toast notification → Done!
Result: 75% faster, zero cognitive load, users can bulk-add games while browsing
